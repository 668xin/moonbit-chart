【大赛硬性约束总规则，编码全程必须遵守】

1. 赛事基础合规规则
   项目归属赛道：上层应用生态，MoonBit 原生可视化图表库 xinxin.moonbit-chart，核心产出可发布到 mooncakes.io 的标准 MoonBit 包。
   代码主体语言：核心业务绘图、图表逻辑、导出工具全部使用 MoonBit 编写；仅辅助开发工具、本地调试脚本可使用 JS/TS/Python，最终参赛验收交付物不允许核心逻辑依赖 JS/TS/Python 运行。
   代码规模约束：最终有效 MoonBit 代码行数控制在 4000~10000 LOC，优先保证测试、文档、边界处理质量，禁止无意义填充冗余代码。

   禁止直接复制现有 MoonBit 生态已成熟、功能高度重合的图表相关项目代码。
   验收强制交付标准（代码必须配套实现以下内容）
   完整 README.md：快速上手、API 文档、示例运行步骤、许可证说明；
   全量单元测试：每个图表组件、工具函数配套测试用例，moon test 一键全部通过；
   CI 自动化配置：支持 moon build、moon test 流水线；
   可运行 Demo 目录：至少 10 个可视化示例，覆盖柱状 / 折线 / 饼图 / 雷达图；
   支持一键打包发布至 mooncakes.io；
   支持输出 SVG 字符串、MD 转 HTML 可视化页面两种产物。
   工作量统计规则：所有有效代码提交记录必须在 2026-04-29 之后，仓库完整保留 Git commit、MR、迭代日志，便于赛事验收核验工作量。

2. MoonBit 代码编码规范强制规则
   模块化分层：严格按照「底层绘图基座→图表组件层→上层命令行工具层→MD 转换工具层」分层，模块职责单一，禁止跨层混乱调用。
   类型规范：全程使用 MoonBit 静态类型，禁止大量any无类型写法；结构体配置统一使用命名参数、默认配置兜底。
   可维护性要求：
   公共函数、结构体、枚举必须添加文档注释；
   复杂算法、坐标换算、颜色处理拆分独立工具模块；
   统一错误处理，不使用裸 panic，提供友好报错信息；
   工程目录规范：固定目录分层，禁止随意创建零散文件；
   性能约束：渲染逻辑做轻量化优化，适配 Wasm 运行环境，避免内存无限制膨胀。
3. JS/TS/Python 辅助脚本限制规则
   JS/TS/Python 仅允许用于：本地预览 SVG 效果、批量生成测试数据、自动生成 Demo 文档、本地调试辅助脚本；
   核心图表渲染、MD 转 HTML 内核逻辑绝不允许使用 JS/TS/Python 实现，仅作为开发辅助工具，验收交付无依赖；
   辅助脚本统一放入 /scripts 文件夹，单独说明用途，不混入核心 MoonBit 业务代码；
   辅助脚本同样开源，配套简单使用说明。
4. 赛事评奖加分编码要求
   支持浅色 / 深色双主题系统；
   图表支持 Tooltip、图例、坐标轴自定义；
   封装命令行二进制工具，可直接读取 Markdown 文件输出完整 HTML 可视化页面；
   完整边界兼容：空数据、超大数值、负数、多组堆叠数据全部做兼容处理；
